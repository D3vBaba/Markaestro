1. Product Overview

Markaestro is a SaaS marketing platform that enables businesses and entrepreneurs to create, launch, and manage paid advertising campaigns across Google Ads, Meta, and TikTok from a single unified dashboard. The platform is subscription-based and targets small to mid-sized businesses who want to run effective paid advertising without switching between multiple ad platforms.

API Usage Summary: Markaestro uses the Google Ads API exclusively on behalf of authenticated end users. All API calls are made using the user's own OAuth access token, scoped to their own Google Ads accounts. Markaestro never accesses, reads, or modifies any Google Ads account without explicit user authorization.

2. End-to-End User Flow

The following describes how a Markaestro user interacts with the Google Ads integration:

1. Connect Account

  1. Connect Account — User authorizes Markaestro via Google OAuth 2.0 using the https://www.googleapis.com/auth/adwords scope. The access token and refresh token are encrypted and stored.
  2. Select Customer — The app calls customers:listAccessibleCustomers to show the user which Google Ads accounts they have access to, and the user selects one.
  3. Create Campaign — User fills in campaign details (name, objective, budget, targeting, keywords, ad creative) using a guided multi-step form.
  4. Launch — Markaestro calls the Google Ads API to create the budget, campaign, ad group, keyword criteria, and responsive search ad in sequence.
  5. Monitor & Manage — User can view performance metrics, pause, resume, update the budget, or delete the campaign from the Markaestro dashboard.

6. Google Ads API Call Sequence (Campaign Launch)

When a user clicks Launch, Markaestro executes the following API calls in sequence against the Google Ads REST API (v23):

# API Endpoint Purpose Status if Failed
1 campaignBudgets:mutate Create daily budget object (amountMicros = dailyBudgetCents × 10,000) Abort – return error to user
2 campaigns:mutate Create campaign with PAUSED status, linked to budget Abort – return error to user
3 campaignCriteria:mutate Add geo targeting criteria (location IDs resolved from country codes) Log warning – continue
4 adGroups:mutate Create ad group (SEARCH_STANDARD or DISPLAY_STANDARD) Abort – return error to user
5 adGroupCriteria:mutate Add keywords as BROAD match criteria (Search campaigns only) Log warning – continue
6 adGroupAds:mutate Create Responsive Search Ad (RSA) with headlines and descriptions Abort – return error to user
7 campaigns:mutate Update campaign status to ENABLED Return partial success with warning

8. Screen: Performance Metrics Sync

Users can sync campaign metrics on demand by clicking the Sync button. Markaestro calls googleAds:searchStream with a GAQL query to retrieve impressions, clicks, cost, conversions, CTR, and average CPC for the campaign lifetime. Metrics are stored in Firestore and displayed in the campaign detail view.

10. Data Handling & Security

Token Storage

OAuth access and refresh tokens are AES-256 encrypted before being stored in Firebase Firestore. Keys are managed via Google Cloud Secret Manager.

Token Refresh

Refresh tokens are used to obtain new access tokens automatically. Failed refreshes are tracked and users are notified to re-authorize.

Workspace Isolation

All data (campaigns, connections, metrics) is scoped by workspace ID. Users can only access their own workspace data.

No Data Resale

Campaign data and performance metrics retrieved from Google Ads are used solely to display information to the authenticated account owner. Data is never sold or shared with third parties.

Secrets Management

The Google Ads developer token and all other credentials are stored in Google Cloud Secret Manager and loaded at runtime. They are never exposed to the client.

OAuth Scope

Only the https://www.googleapis.com/auth/adwords scope is requested, along with openid, email, and profile for user identification.
